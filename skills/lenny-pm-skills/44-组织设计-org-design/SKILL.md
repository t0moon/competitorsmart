# 组织设计

## 何时使用此技能

当你需要：
- 设计或重组产品组织架构
- 在职能型和事业部型组织之间做选择
- 简化管理层级和决策流程
- 平衡用户体验一致性和团队速度

## 核心原则

### 1. 职能型 vs 事业部型模型
职能型组织（如Apple）集中控制，用户体验更一致；事业部型组织（如Amazon）最小化依赖，团队并行速度更快。

### 2. 回归创业状态
去除不做实际工作的"人员经理"层级，回到每个管理者都深入了解工作细节的创业状态。

### 3. 架构决定产品
组织架构深刻影响产品的用户体验和发展速度，需要根据产品策略选择合适的模型。

## 专家洞察

### Brian Chesky
> "我们转向了职能型模型。我们回到了创业公司的状态。"

**核心洞察**：讨论了从事业部型到职能型模型的根本转变，去除管理层级，重组角色以消除不了解工作的"人员经理"。

**如何应用**：
- 考虑职能型组织以获得更好的协调和一致性
- 去除不深入了解工作的管理层级
- 回归创业公司的扁平结构

### Gustav Soderstrom
> "一端是像Amazon这样的...最小化依赖，所以你可以并行运行...另一端是像Apple这样的...由接近单一个体的东西集中组织。"

**核心洞察**：提供了深入的框架来比较集中型（Apple）与分散型（Amazon）模型，以及它们如何影响用户体验和速度。

**如何应用**：
- 理解集中型模型有利于一致的用户体验
- 理解分散型模型有利于团队速度和自主性
- 根据产品策略选择合适的组织模型

## 常见错误

- 盲目选择组织模型而不考虑产品策略
- 保留不了解实际工作的管理层级
- 在需要一致用户体验时选择过度分散的模型
- 在需要快速并行时选择过度集中的模型

## 关键战术

| 战术 | 说明 |
|------|------|
| **模型评估** | 根据产品策略评估职能型vs事业部型 |
| **层级精简** | 去除不了解工作细节的管理层级 |
| **依赖分析** | 分析团队间的依赖，决定集中或分散程度 |
| **用户体验一致性** | 评估需要多大程度的跨团队协调 |
| **速度优先级** | 评估并行开发速度的重要性 |
