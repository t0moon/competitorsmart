# 书面沟通

## 何时使用此技能

当你需要：
- 撰写清晰的内部文档
- 通过写作理清思路
- 规模化传递信息
- 建立项目的权威信息源

## 核心原则

### 1. 聚焦"如何做"
有效的沟通聚焦于"如何做"——具体的应用和细节——而非证明"是什么"或"为什么"。

### 2. 写作即思考
写作是整理散乱思维、强制思维清晰的"自我疗愈"。

### 3. 清晰可规模化
写作是 PM 规模化传递清晰度和展示对客户声音深刻理解的主要工具。

## 专家洞察

### Wes Kao
> ""超级具体的如何"这个概念是指大多数作者、大多数课程讲师在"是什么"和"为什么"上花了太多时间，而在"如何做"上花的时间不够。所以如果你想想读你文章的人，他们大多可能已经同意你所说的一般前提。"

**核心洞察**：有效的沟通聚焦于"如何做"——具体的应用和细节——而非证明"是什么"或"为什么"。

**如何应用**：
- 最小化花在一般前提或背景上的时间
- 聚焦于如何应用一个想法的具体步骤、细节和例子

### Julie Zhuo
> "我当时这样处理写作，现在仍然如此——把它当作写给自己的信。这是框架。这是我需要给自己的建议，我需要去真正做得更好的事情，这就是写作对我的意义，它对理清我的思路非常有帮助。"

**核心洞察**：写作是一种"自我疗愈"，能整理散乱的思维并强制思维清晰。

**如何应用**：
- 写"给自己的信"来理清你对复杂话题的看法
- 用写作作为工具来让自己更自如地在大型会议上发言

### Kevin Yien
> "写作是规模化的清晰度，而 PM 工作的一个关键组成部分是在内部和外部创造清晰度...如果你不能销售或支持你自己的产品，我不信任你能构建产品。所以这就是我认为写作是基础组成部分的地方。"

**核心洞察**：写作是 PM 规模化传递清晰度和展示对客户声音深刻理解的主要工具。

**如何应用**：
- 用你试图服务的人的声音写出有说服力的信息
- 通过清晰的 PRD 聚焦内部清晰度，通过可用于支持/销售的写作聚焦外部清晰度

### Naomi Gleit
> "我真的相信框架可以驱动极致的清晰度。我在很多不同的项目上工作。很多时候我在启动一个新项目时，会问'我在哪里能学到我需要了解的这个项目的信息？'我问五个不同的人，得到五个不同的答案。那是..."

**核心洞察**：每个项目都应该有一个单一的"权威文档"作为真理的中心来源，并链接到所有其他相关文档。

**如何应用**：
- 为每个项目创建一个权威文档
- 确保文档链接到所有其他相关文档

### Boz
> "沟通就是工作...它完全是通过创建影响其他人的工件或言语表达来完成的。这就是全部。"

**核心洞察**：领导力本质上是一项沟通任务；如果你的想法没有突破或没有被理解，你就没有完成工作。

**如何应用**：
- 对你的信息是否被听众成功接收承担极端的责任
- 理解沉默和不检查也是一种沟通形式

## 常见错误

- 在"是什么"和"为什么"上花太多时间
- 项目没有单一的权威信息源
- 不把写作作为理清思路的工具
- 忽视沟通是领导的本质工作

## 关键战术

| 战术 | 说明 |
|------|------|
| **聚焦如何做** | 减少背景，增加具体步骤和例子 |
| **权威文档** | 每个项目一个中心文档 |
| **给自己的信** | 通过写作理清复杂思路 |
| **客户声音** | 用服务对象的语言写作 |
