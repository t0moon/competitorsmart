# 复盘回顾

## 何时使用此技能

当你需要：
- 项目结束后进行系统性总结
- 从失败中提取学习和改进点
- 建立团队的学习和反思文化
- 评估OKR或目标完成情况

## 核心原则

### 1. 预设"终止标准"
事前分析（Pre-mortem）只有在设定了"终止标准"时才有效——即预先确定会触发转型或终止的信号。

### 2. 将复盘重新定义为学习
把"事后分析"（Post-mortem）改称为"回顾"（Retrospective），将焦点从责备转移到学习，有助于正常化失败并鼓励冒险。

### 3. 制度化学习分享
建立"学习回顾"仪式，确保实验数据真正被用于未来决策，并在整个公司传播洞察。

## 专家洞察

### Annie Duke
> "所以事前分析只有在你设定了终止标准时才有效。承诺如果你看到这些信号就采取行动。"

**核心洞察**：事前分析只有在产生"终止标准"时才有效——即预先确定会触发转型或终止的信号。

**如何应用**：
- 在事前分析中识别项目失败的早期信号
- 预先承诺如果出现这些信号就采取特定行动（如终止项目）

### Eeke de Milliano
> "与其称之为事后分析，不如称之为回顾，这样它就是一件积极的事情。比如，'嘿，我们从这件事中学习。'"

**核心洞察**：将事后分析重新定义为回顾有助于正常化失败，让团队专注于学习而非责备。

**如何应用**：
- 将"事后分析"重命名为"回顾"
- 用光照亮失败，以减轻大胆尝试的恐惧

### Ben Williams
> "我们有这些团队级别的影响和学习回顾...团队持续记录从数据探索、实验、用户研究等获得的任何学习。他们在每周的影响和学习文档中记录...大部分时间花在讨论已经..."

**核心洞察**：制度化"学习回顾"仪式确保实验数据真正被用于未来决策。

**如何应用**：
- 举办专注于洞察而非状态更新的每周"影响和学习"回顾
- 在整个公司传播学习内容，提升其他团队

### Carole Robin
> "这个缩写是A-F-O-G，又一个F-ing...又一个该死的成长机会。每个上过我课的学生...都知道这个缩写，因为当事情出错或一个人经历失败时，我的问题总是：那你学到了什么？"

**核心洞察**：将失败重新定义为学习机会对于长期成长和韧性至关重要。

**如何应用**：
- 当失败发生时，立即问："这里的教训是什么？"
- 使用AFOG缩写在痛苦的挫折中保持视角

### Christina Wodtke
> "重要的是，为什么是80%？真正专注于学习...一切都在于回顾。确保你的评分次于回顾，这是我要说的最重要的事情，因为那才是有价值的。"

**核心洞察**：评估OKR的价值在于对为何达成或未达成目标的回顾分析，而非数字本身。

**如何应用**：
- 优先关注分数背后的"为什么"，而非数字的精确度
- 使用季度末回顾来识别阻止达到100%的系统性障碍

## 常见错误

- 进行事前分析但不设定终止标准
- 将复盘变成责备会议，而非学习机会
- 记录学习但不传播或应用到未来决策
- 过于关注OKR的分数而非背后的原因

## 关键战术

| 战术 | 说明 |
|------|------|
| **终止标准** | 在事前分析中预设触发转型或终止的信号 |
| **重命名为回顾** | 将"事后分析"改称"回顾"，聚焦学习 |
| **每周学习回顾** | 举办专注洞察而非状态的定期回顾 |
| **AFOG思维** | 将失败视为"又一个成长机会" |
| **为什么分析** | 分析目标完成背后的原因，而非仅看数字 |
