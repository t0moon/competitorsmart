# 用户引导

## 何时使用此技能

当你需要：
- 设计新用户的首次产品体验
- 优化从注册到激活的转化漏斗
- 缩短用户达到"啊哈时刻"的时间
- 借鉴游戏设计原则优化引导流程

## 核心原则

### 1. 引导即产品
引导不是产品的附属，而是产品体验的起点。从引导体验开始设计产品。

### 2. 前30秒至关重要
产品的前30秒决定用户是否继续探索，要让这30秒感觉神奇。

### 3. 早期体验驱动留存
留存的最大收益往往来自改变早期用户体验，而非后续功能改进。

## 专家洞察

### Cam Adams
> "我们在上线前的最后几个月花了大量时间优化引导流程，这真的很关键，因为产品功能已经到位了...但有一些东西阻碍了用户。"

**核心洞察**：即使产品功能完备，设计一个能引导用户到达"啊哈时刻"的首次使用体验仍然是决定性的。

**如何应用**：
- 在产品功能完成后，专门投入时间优化引导流程
- 识别并移除阻碍用户体验核心价值的障碍
- 将引导视为产品成功的关键因素，而非次要细节

### Dan Hockenmaier
> "我认为留存方面最大的收益往往来自改变早期用户体验。"

**核心洞察**：早期用户体验的优化是提升留存最大的杠杆，这是一个独特的产品设计领域。

**如何应用**：
- 优先投资优化前几次用户会话的体验
- 追踪早期体验指标与长期留存的相关性
- 将早期体验优化作为增长团队的核心职责

### Grant Lee
> "我们会尽一切可能让产品的前30秒感觉神奇。"

**核心洞察**：产品的"前30秒"和"第一英里"是口碑增长的主要驱动力。

**如何应用**：
- 设计让用户在前30秒就感受到价值的体验
- 将首次体验的"神奇感"作为核心设计目标
- 观察新用户的前几次操作，优化每一个步骤

### Merci Grace
> "我对引导的思考真的源于我设计游戏的经验，在那里我会从引导体验开始设计整个游戏。"

**核心洞察**：从游戏设计原则出发设计产品引导，避免轮播图或非原生框架等常见错误。

**如何应用**：
- 借鉴游戏设计的渐进式教学原则
- 避免使用静态轮播图来解释产品
- 让用户通过实际操作来学习，而非阅读说明
- 确保引导体验与产品的原生体验一致

## 常见错误

- 用静态轮播图或弹窗解释产品功能
- 让用户在体验价值前填写大量信息
- 引导流程与实际产品体验脱节
- 忽视前30秒体验的重要性
- 将引导设计推迟到产品开发末期

## 关键战术

| 战术 | 说明 |
|------|------|
| **30秒法则** | 让用户在30秒内感受到产品的核心价值 |
| **渐进式教学** | 借鉴游戏设计，通过实际操作逐步教学 |
| **障碍识别** | 找出阻碍用户体验价值的具体步骤 |
| **啊哈加速** | 缩短用户到达啊哈时刻的路径 |
| **原生体验** | 保持引导与产品核心体验的一致性 |
