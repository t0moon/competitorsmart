# 技术路线图

## 何时使用此技能

当你需要：
- 制定技术团队的战略规划
- 对齐技术投资与业务目标
- 与利益相关者沟通技术方向
- 建立可改进的技术战略文档

## 核心原则

### 1. 写下来才能改进
书面的战略是组织对齐的必要条件，提供了可以被批评和改进的基准。

### 2. 使用结构化框架
采用诊断、指导原则和行动的框架来组织技术战略。

### 3. 不完美也要写
即使不是特别有说服力的战略文档，也比没有文档更有价值。

## 专家洞察

### Will Larson
> "战略的第一条规则是：如果你把它写下来，你就可以改进它。如果没有写下来，很难说这个PM只是不是一个好PM，还是他们在尝试应用他们误解的战略……如果你有一份书面文档，即使它不是超级有说服力……"

**核心洞察**：书面的战略是组织对齐的必要条件，提供了可以被批评和改进的基准。

**如何应用**：
- 将战略写下来，即使不完美，也能允许"调试"和对齐
- 使用Richard Rumelt框架：诊断、指导原则和行动

## 常见错误

- 只在脑中有战略而不写下来
- 战略文档过于模糊没有可执行性
- 不定期更新和回顾技术路线图
- 技术路线图与业务目标脱节
- 没有清晰的诊断和指导原则

## 关键战术

| 战术 | 说明 |
|------|------|
| **写下来** | 将技术战略形成书面文档 |
| **Rumelt框架** | 使用诊断、指导原则、行动的结构 |
| **定期回顾** | 定期更新和审视技术路线图 |
| **对齐验证** | 确保技术投资与业务目标对齐 |
