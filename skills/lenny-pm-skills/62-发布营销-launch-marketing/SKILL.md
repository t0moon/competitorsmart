# 发布营销

## 何时使用此技能

当你需要：
- 策划新产品或新功能的发布
- 设计能引发关注的发布活动
- 与媒体合作进行发布报道
- 确保内部团队为发布做好准备

## 核心原则

### 1. 独家策略
对于早期创业公司，与单一媒体建立独家合作通常比广撒网的禁令发布更有效。

### 2. 亮点功能制造病毒
成功的发布往往需要一个"亮点"功能——即使它不是核心用例，也能吸引人们注意到产品。

### 3. 就绪度比时间表更重要
成功的产品发布需要确保内部团队知道如何谈论和支持新功能，而不仅仅是何时发布。

## 专家洞察

### Arielle Jackson
> "对于早期创业公司，我们几乎总是将发布公告作为独家新闻来运作，这意味着你把新闻给一个单一的媒体。他们是唯一能写这个故事的人。"

**核心洞察**：现代创业公司的PR通常需要向单一媒体提供独家新闻，而非广泛的禁令发布。

**如何应用**：
- 确定一个覆盖你这个阶段公司的目标媒体
- 在发布公告前准备好网站以迎接流量

### Bret Taylor
> "有了卫星图像，老实说这不是Google Maps最重要的部分，但它是牛排上的油花，它创造了...一个病毒时刻。"

**核心洞察**：成功的发布往往需要一个"亮点"功能——视觉上令人印象深刻或具有病毒性——来吸引人们注意核心功能。

**如何应用**：
- 确定一个能产生话题的"亮点"功能，即使它不是主要用例

### Emilie Gerber
> "当我们推销融资故事时，我们会非常清楚地说明，'嘿，我们正在联系你关于一个创业公司发布或融资。我们将其作为独家新闻提供给你，意味着如果你接受这个故事，我们不会再联系其他人。这是筹集的金额，这是所在的领域...这是所在的领域...'"

**核心洞察**：融资公告应该作为独家新闻来推销，提供清晰、高层次的细节，而非复杂的叙事。

**如何应用**：
- 一次只向一个记者提供独家故事
- 在初始推销中包含筹集金额、行业领域和领投方

### Marc Benioff
> "我把所有东西都往墙上扔。我在看什么会粘住。我在试图找到获胜的策略并把它变成获胜的战略。"

**核心洞察**：成功的发布往往涉及大量实验，以识别一个可以扩展为更广泛战略的突破性策略。

**如何应用**：
- 同时测试多种营销策略
- 识别哪个特定行动"粘住"或产生病毒效应
- 加倍投入获胜策略以形成核心战略

### Christine Itwaru
> "有'知道某事即将到来'和'知道该如何处理它'之间的区别。你可以用某个状态跟踪工具说，'这个功能在Q4到来'...但缺少的是'该如何处理它'以及如何定位它、如何谈论它以及所有这些事情。"

**核心洞察**：成功的产品发布需要"就绪度"——确保内部团队知道如何谈论和支持新功能，而不仅仅是何时发布。

**如何应用**：
- 超越简单的状态更新，为内部团队提供"就绪度"指南
- 确保每次重大发布都包含如何向客户定位新价值的具体说明

## 常见错误

- 广撒网而非建立独家媒体关系
- 发布时没有一个能制造话题的"亮点"功能
- 只告诉团队"何时"发布而非"如何"谈论
- 过早确定单一策略而非大量实验
- 网站和支持团队没有为流量做好准备

## 关键战术

| 战术 | 说明 |
|------|------|
| **独家合作** | 与单一媒体建立独家报道关系 |
| **亮点功能** | 识别能制造病毒传播的视觉或话题功能 |
| **就绪度指南** | 为内部团队提供如何定位和谈论新功能的指导 |
| **策略实验** | 大量测试不同营销策略，找到获胜方案 |
| **网站准备** | 确保网站和基础设施准备好迎接发布流量 |
