# 平台策略

## 何时使用此技能

当你需要：
- 制定平台产品的战略方向
- 设计内部平台以提升开发者生产力
- 理解平台生命周期和开放/关闭策略
- 识别具有网络效应的生态系统机会

## 核心原则

### 1. 平台即产品
内部平台应被视为产品，配备专门的产品管理，专注于用户（开发者）生产力。

### 2. 平台四阶段生命周期
平台遵循可预测的四阶段生命周期：市场条件满足、建立护城河、开放平台、关闭以实现变现。

### 3. 寻找"松露"
平台和生态系统的成功来自识别"松露"——参与者越多价值越大的复合循环动态。

## 专家洞察

### Camille Fournier
> "平台工程不仅仅是维护云基础设施...平台是产品，最终。你应该考虑如何创建连贯的产品，使这家公司更有生产力？"

**核心洞察**：内部平台应被视为产品，配备专门的产品管理，专注于用户（开发者）生产力。

**如何应用**：
- 为内部平台团队招聘产品经理
- 关注基于结果的指标，如减少周期时间或解决扩展瓶颈

### Brian Balfour
> "四个步骤本质上是，一是我称之为零步骤。是市场条件已经满足。第一步是关于护城河，第二步是关于平台开放，第三步是关于平台关闭以实现控制和变现。"

**核心洞察**：平台遵循可预测的四阶段生命周期，从市场共识到关闭以实现变现。

**如何应用**：
- 识别为平台所有者创造防御性的护城河
- 建立具有对开发者明确激励的第三方平台
- 预期关闭阶段，平台会优先考虑自己的变现和第一方应用

### Jeremy Henrickson
> "你能烘焙到清晰平台中的越多，它就越能减少每个在领域部分问题上工作的人的决策复杂性。"

**核心洞察**：具有清晰接口的定义良好的平台通过减少各个团队的认知负担来简化产品开发。

**如何应用**：
- 投资于系统思维，创建简化特定领域工作空间的平台
- 基于产品需求迭代平台，而非在真空中构建

### Alex Komoroske
> "任何形状像生态系统、有某种网络效应的东西...你必须知道你在寻找什么，找到如果它有效就会以加速速度发展的事物的动态。很多很多事情本质上都有这种形状。"

**核心洞察**：平台和生态系统的成功来自识别"园艺"机会——具有内在复合循环、会自行增长的项目。

**如何应用**：
- 寻找"松露"——参与者越多价值越大的动态
- 只有在收到效用和采用信号时才逐步投资生态系统项目

### Aparna Chennapragada
> "我思考我们如何定位以及我们用GitHub做什么的方式是...所以它是一个系统，不仅仅是一个产品或一组功能...你需要有一个系统。你需要有一种能力，一个完整的瑞士军刀工具包，不仅仅是自动完成，不仅仅是聊天，不仅仅是..."

**核心洞察**：长期平台防御性来自构建一个全面的系统和上下文存储库，而非仅仅是单一功能或工具。

**如何应用**：
- 构建捕获用户上下文和存储库的系统以创造防御性
- 关注复杂专业工作流的"瑞士军刀工具包"方法
- 将多个AI能力（自动完成、聊天、代理）整合到统一系统中

## 常见错误

- 将内部平台视为基础设施而非产品
- 不理解平台生命周期的各个阶段
- 在没有验证信号的情况下过度投资生态系统
- 只构建单一功能而非全面系统

## 关键战术

| 战术 | 说明 |
|------|------|
| **平台产品化** | 为内部平台配备PM，关注开发者生产力 |
| **生命周期意识** | 理解平台的开放、关闭、变现阶段 |
| **松露识别** | 寻找具有网络效应和复合增长的机会 |
| **渐进投资** | 收到效用信号后才逐步加大投资 |
| **系统构建** | 构建全面系统而非单一功能 |
