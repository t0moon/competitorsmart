# 设计工程

## 何时使用此技能

当你需要：
- 理解设计工程这个新兴角色
- 组建能快速原型和交付的团队
- 在AI时代评估技能组合
- 提升设计到实现的转化效率

## 核心原则

### 1. 设计与工程的融合
设计工程是一个融合UX设计和工程的特定职能，能够更快地原型和交付。

### 2. AI时代的高价值技能
结合高端设计感知力和交付生产级代码能力的角色在AI驱动开发中备受重视。

### 3. 缩短从想法到实现的距离
设计工程师可以独立完成从设计到实现的完整循环。

## 专家洞察

### Gaurav Misra
> "我最终加入了设计团队……我在那里做的很大一部分工作是创建这个叫做设计工程的职能。"

**核心洞察**：这位专家讨论了在Snap和Captions创建特定"设计工程"职能的过程，该职能融合了UX设计和工程以更快地原型和交付。

**如何应用**：
- 考虑在团队中设立设计工程角色
- 让设计工程师负责快速原型和验证
- 缩短设计到实现的反馈循环

### Guillermo Rauch
> "我们在Vercel博客上发布的一件让人兴奋的事是关于设计工程的，因为我们注意到很多在Vercel非常成功的人是那些同时具备设计和工程技能的人。"

**核心洞察**：这位专家强调了一个特定的新兴角色，它结合了高端设计感知力和交付生产级代码的能力，在AI驱动开发中备受重视。

**如何应用**：
- 在招聘时寻找同时具备设计和工程能力的人才
- 鼓励设计师学习编码或工程师提升设计能力
- 将设计工程作为团队能力建设的方向

## 常见错误

- 将设计和工程完全分离
- 忽视融合型人才的价值
- 过长的设计到实现周期
- 不投资培养跨界能力
- 低估AI时代对这类角色的需求

## 关键战术

| 战术 | 说明 |
|------|------|
| **设立角色** | 在团队中正式设立设计工程角色 |
| **快速原型** | 让设计工程师负责概念验证 |
| **跨界培养** | 鼓励设计师学编码或工程师学设计 |
| **缩短循环** | 用设计工程缩短从想法到实现的周期 |
