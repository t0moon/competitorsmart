# 逆向工作法

## 何时使用此技能

当你需要：
- 使用亚马逊的 PR/FAQ 方法
- 从客户需求出发定义产品
- 避免"手边有什么做什么"的陷阱
- 进行长期产品规划

## 核心原则

### 1. 从问题开始
逆向工作法的核心是从问题开始，痴迷于问题，让问题引导你走向解决方案。

### 2. 客户优先
从客户需求出发，倒推解决方案，相信业务结果会随客户满意度而来。

### 3. 愿景驱动规划
有效的规划从对未来状态的清晰愿景开始，然后逆向推导到达的步骤。

## 专家洞察

### Ian McAllister
> "逆向工作法是关于问题的，从那里开始，痴迷于问题，让它引导你进入解决方案。所以做错的团队就是没这样做。他们没有逆向工作。他们有一些想要构建的东西...如果你说我们可以而且..."

**核心洞察**：逆向工作法中最常见的失败是将问题硬套到预先存在的解决方案上，而非从客户痛点开始。

**如何应用**：
- 在定义解决方案前先写问题段落
- 避免仅仅因为"厨房里有这些食材"就组合它们；确保它们解决一个具体问题

### Bill Carr
> "Jeff 会说，我们把这当作信条。如果我们服务好客户，如果我们优先考虑客户并为他们交付，像销售额、收入、活跃客户、股价和自由现金流这样的东西就会随之而来。所以因此，当我们做决定时..."

**核心洞察**：从客户需求出发，倒推解决方案，相信业务结果会随客户满意度而来。

**如何应用**：
- 每个决定都从问"什么对客户最好"开始
- 从理想的客户解决方案倒推来确定必要的工程和资源需求

### Anuj Rathi
> "本质上你逆向工作的是在某一天运转的整个机器，为了...对于 GTM 的日期，我们需要从这里到那一天做什么才能让 GTM 成功，而且还有，我们会创造什么机器让这个..."

**核心洞察**：逆向工作法是关于将整个组织机器和 GTM 策略对齐到一个具体的发布日期，而不仅仅是写一个客户引用。

**如何应用**：
- 使用 PRFAQ 来识别可能破坏项目的潜在内部联盟或利益相关者
- 写三个发散的、完整思考的新闻稿来帮助领导层比较替代路线
- 为特定组织需求如合规、法务或市场影响包含必要的 FAQ

### Christopher Lochhead
> "他称之为回溯（backcasting）以区别于预测（forecasting）...站在那个未来，五年后，回望现在，我们做了什么让这个不同的未来发生？这就是品类设计。这就是你如何把自己从过去解放出来。"

**核心洞察**：回溯让创业者能够设想一个激进的未来，而不受当前限制的约束。

**如何应用**：
- 设想你的公司五年后的样子，然后倒推确定到达那里需要的步骤
- 拒绝当前事情如何做的前提来打开新的可能性

### Daniel Lereya
> "我如何想象公司和产品在一个季度后对我们的客户会有什么不同和更好？然后从那里倒推。"

**核心洞察**：有效的规划从对未来状态的清晰愿景开始，然后逆向推导到达的步骤。

**如何应用**：
- 创建一张幻灯片描述一年后客户的生活有何不同
- 使用未来状态描述来识别哪些当前项目真正有影响力

## 常见错误

- 先有解决方案再找问题
- PRFAQ 只写客户引用而忽略组织对齐
- 被当前限制束缚而无法想象激进的未来
- 用预测代替回溯

## 关键战术

| 战术 | 说明 |
|------|------|
| **问题段落优先** | 在解决方案前明确客户痛点 |
| **多版本 PRFAQ** | 写 3 个不同版本帮助比较 |
| **回溯法** | 从 5 年后的愿景倒推 |
| **未来状态描述** | 一张幻灯片描述客户生活的改变 |
