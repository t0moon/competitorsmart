# 辅导PM

## 何时使用此技能

当你需要：
- 发展和提升团队中PM的能力
- 诊断PM在哪些技能层面需要提升
- 建立PM成长和发展的系统框架
- 扩展产品组织的规模

## 核心原则

### 1. 教练是领导者的首要职责
在顶级公司（Google、Apple、Microsoft），教练是产品领导者的首要职责，是唯一有效扩展产品组织的方式。

### 2. 使用教育框架诊断技能
使用布鲁姆分类法等教育框架来诊断和提升PM技能：知识、理解、应用、分析、综合和评估。

### 3. 五步发展框架
有效的PM发展需要：定义"好"的标准、评估现状、共同愿景、发展计划和跟进。

## 专家洞察

### Bangaly Kaba
> "布鲁姆分类法描述了你需要掌握某事的不同层次或批判性思维顺序...知识，然后是理解，然后是应用、分析、综合和评估。"

**核心洞察**：讨论了使用布鲁姆分类法等特定教育框架来诊断和提升PM技能，这与一般的团队文化不同。

**如何应用**：
- 使用布鲁姆分类法评估PM在哪个层次
- 针对不同层次设计不同的发展方法

### Marty Cagan
> "Google的Sundar一直在说，他们在领导者身上寻找的第一件事是优秀的教练。"

**核心洞察**：强调在顶级公司（Google、Apple、Microsoft），教练是产品领导者的首要职责，是唯一有效扩展产品组织的方式。

**如何应用**：
- 将教练作为领导角色的核心职责
- 通过教练而非直接管理来扩展组织

### Petra Wille
> "第一步是真正有一个在你的环境中好的产品人是什么样子的明确定义。"

**核心洞察**：概述了发展PM的具体5步框架（好的定义、当前评估、共同愿景、发展计划和跟进），这与一般的领导力不同。

**如何应用**：
- 首先定义在你的环境中"好的PM"是什么样子
- 评估当前状态
- 建立共同愿景
- 制定发展计划
- 定期跟进

## 常见错误

- 将管理与教练混为一谈
- 没有定义"好的PM"在你的环境中是什么样子
- 跳过诊断，直接给出通用建议
- 缺乏系统的跟进机制

## 关键战术

| 战术 | 说明 |
|------|------|
| **布鲁姆分类法** | 用知识-理解-应用-分析-综合-评估层次诊断技能 |
| **好的定义** | 定义在你的环境中好的产品人是什么样子 |
| **当前评估** | 客观评估PM当前的能力水平 |
| **共同愿景** | 与PM共同建立发展愿景 |
| **发展计划+跟进** | 制定具体计划并定期跟进进展 |
