# 不确定性规划

## 何时使用此技能

当你需要：
- 在高度不确定的环境中制定计划
- 管理实验和假设验证流程
- 应对危机或快速变化的情况
- 在信息不完整时做出决策

## 核心原则

### 1. 重视学习而非输赢
健康的实验文化更看重从"失败"的冒险押注中学到的东西，而非安全的渐进式"胜利"。实验的本质是学习，而非证明对错。

### 2. 数据是指南针而非GPS
数据应该用来验证或否定直觉，而非作为决策的确定性路线图。等待数据给出确切答案往往意味着速度太慢或方向错误。

### 3. 为混乱预留缓冲
在混乱的环境中，规划必须包含明确的时间缓冲和应急计划。根据范围大小，缓冲可以从几天到几个月不等。

## 专家洞察

### Ramesh Johari
> "在科学史上，实验从来不是关于输赢的...实验总是非常假设驱动的。关键是你学到了什么？这是一个非常重要的区别，因为这意味着如果我选择了一个大的、有风险的东西，它'失败'了，意思是..."

**核心洞察**：健康的实验文化更看重从"失败"的冒险押注中学到的东西，而非安全的渐进式"胜利"。

**如何应用**：
- 为每个实验定义清晰的假设，这样"失败"仍然能提供可操作的商业洞察
- 避免仅根据每季度"胜利"数量来评判数据科学家，以防止风险规避

### Alex Hardimen
> "我会想到战时产品管理，对吧？你进入其中，我认为需要这种令人难以置信的谦逊，真正理解并首先诊断平台上实际发生了什么。"

**核心洞察**：在危机中管理需要"战时"的谦逊，在尝试解决方案之前准确诊断问题。

**如何应用**：
- 在危机中优先诊断而非立即行动
- 以谦逊的态度对待未知的平台问题

### Nikita Bier
> "开发一个可重复的测试流程，这实际上比任何其他因素更能影响你成功的概率。消费产品创意是否有效是如此不可预测。"

**核心洞察**：消费产品的成功取决于测试流程的质量和速度，而非最初的创意。

**如何应用**：
- 建立一个系统来"多次尝试"，以降低不可预测性的风险

### Shaun Clowes
> "数据更像是指南针而非GPS。如果你把数据当作给你答案的东西，你总是错的。总是错或者慢。错或慢或两者兼有，因为大多数情况下数据不会给你答案。它只是告诉你，你刚才说的是否荒谬，或者有..."

**核心洞察**：数据应该用来验证或否定直觉，而非作为决策的确定性路线图。

**如何应用**：
- 使用数据来否定你的假设，而不是等待它告诉你确切该做什么

### Upasna Gautam
> "我们总是需要有A，当然是转型的能力，但也要在这些场景中有备份和缓冲。所以每次我们规划时，都会为每天发生的所有混乱建立缓冲。"

**核心洞察**：在混乱的环境中，规划必须包含明确的时间缓冲和应急计划才能保持有效。

**如何应用**：
- 根据范围大小，在项目时间表中建立从几天到几个月不等的缓冲
- 每天客观评估情况，决定是使用缓冲还是进入下一阶段

## 常见错误

- 仅根据"胜利"来评判实验，而非学到的东西
- 等待数据给出确切答案，导致行动太慢
- 规划时不预留混乱和意外的缓冲时间
- 在危机中急于行动，而非先准确诊断问题

## 关键战术

| 战术 | 说明 |
|------|------|
| **假设驱动实验** | 为每个实验定义清晰假设，让"失败"也有学习价值 |
| **可重复测试流程** | 建立系统化的测试机制，多次尝试以降低风险 |
| **数据作为指南针** | 用数据验证或否定假设，而非等待确切答案 |
| **时间缓冲** | 根据项目范围预留适当的缓冲时间 |
| **战时诊断** | 危机中先谦逊诊断问题，再尝试解决方案 |
