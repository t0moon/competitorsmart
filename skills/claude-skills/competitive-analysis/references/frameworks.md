## 分析框架说明

### PEST 宏观环境分析

从四个维度审视行业所处的外部环境，帮助理解行业发展的大背景：

- **Political（政治/政策）**：行业监管政策、政府扶持/限制、法规变化、贸易政策
- **Economic（经济）**：宏观经济走势、行业市场规模、资本市场态度、上下游成本变化
- **Social（社会）**：用户习惯变迁、人口结构变化、消费观念演变、行业人才供给
- **Technological（技术）**：技术成熟度、新技术冲击、基础设施变化、数字化程度

使用要点：不要罗列新闻，要分析每个因素对行业竞争格局的具体影响。

### 商业模式画布（Business Model Canvas）

用 9 个要素完整描述一个公司如何创造、传递和获取价值：

| 要素 | 核心问题 |
|------|---------|
| 客户细分 | 为谁服务？最重要的客户是谁？ |
| 价值主张 | 为客户解决什么问题？提供什么独特价值？ |
| 渠道通路 | 通过什么渠道触达客户？ |
| 客户关系 | 与客户建立什么类型的关系？ |
| 收入来源 | 靠什么赚钱？客户愿意为什么付费？ |
| 核心资源 | 商业模式运转需要什么关键资源？ |
| 关键业务 | 必须做好哪些关键活动？ |
| 重要伙伴 | 关键供应商和合作伙伴是谁？ |
| 成本结构 | 最大的成本项是什么？ |

对比分析时重点关注：价值主张的差异化、收入来源的多元性、核心资源的壁垒高度。

### KANO 模型（功能分类）

将产品功能按用户感知分为三类，用于功能对比时的分类依据：

- **基本型需求（Must-have）**：没有会不满，有了觉得理所当然。是行业入场的基本门槛。
- **期望型需求（Performance）**：做得越好满意度越高，是竞争的主战场。
- **兴奋型需求（Delighter）**：用户没期望但提供了会惊喜，是差异化的来源。

对比分析时重点关注：
- 我方在基本型需求上有没有缺失（短板）
- 期望型需求上谁做得更好（竞争力）
- 兴奋型需求上谁有独特功能（差异化机会）

### SWOT 分析

综合评估竞争态势的经典框架：

- **Strengths（优势）**：我方相对竞品的长处
- **Weaknesses（劣势）**：我方相对竞品的短板
- **Opportunities（机会）**：外部环境中的有利因素
- **Threats（威胁）**：外部环境中的不利因素

分析时的关键：S/W 是内部对比得出的，O/T 是外部环境决定的。最有价值的是交叉策略——用 S 抓 O、用 S 化解 T、针对 W 弥补或规避。

### UX 五层模型（产品架构分析）

从战略到表现逐层拆解产品：

1. **战略层**：产品目标 + 用户需求
2. **范围层**：功能规格 + 内容需求
3. **结构层**：交互设计 + 信息架构
4. **框架层**：界面设计 + 导航设计 + 信息设计
5. **表现层**：视觉设计

竞品分析中主要用于产品架构对比，重点看信息架构（功能如何组织）和交互流程（核心路径是否顺畅）。
